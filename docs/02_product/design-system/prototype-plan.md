# Prototype Plan - Design System Validation

新デザインシステム（Refined Modern SaaS）の視覚的検証のためのプロトタイプ計画。

## 目的

- 新しいカラーパレット（Warm Gray + Purple）の視覚的確認
- アニメーション（Purposeful Animation）の動作検証
- ユーザーフィードバックの収集

## プロトタイプツール選定

### 選択肢の比較

| 項目 | Figma | Next.js コードプロトタイプ |
|------|-------|--------------------------|
| 速度 | 高速（1-2時間） | 中速（4-8時間） |
| 忠実度 | 中（静的モックアップ） | 高（実際のコンポーネント） |
| アニメーション | プロトタイプモードで可能 | Framer Motion で完全再現 |
| 再利用性 | 低（本番に統合できない） | 高（そのまま本番利用可） |
| 技術検証 | 不可 | 可（パフォーマンス、依存関係） |
| コスト | 低 | 中 |

### 推奨: Next.js コードプロトタイプ

**理由:**
1. **既存コードベースとの統合**: 既に実装されているコンポーネントをベースに作成可能
2. **技術検証**: Framer Motion、@dnd-kit/core の実際の動作を確認できる
3. **再利用性**: プロトタイプがそのまま本番コードになる
4. **リアルな動作**: 60fps アニメーション、GPU アクセラレーションを実際に検証できる

**デメリット:**
- 初期コストが高い（Figma の 2-4倍の時間）
- デザイナーの直接編集が困難

## プロトタイプ対象画面

### 1. Dashboard (ダッシュボード)

**検証項目:**
- カード配置とレイアウト
- KPI 数値の視認性（JetBrains Mono フォント）
- 品質スコアのカラーリング（Emerald/Amber/Red）
- ホバー時のアニメーション

**コンポーネント:**
- `Header` (Sidebar 統合)
- `KPICard` (数値表示)
- `QualityScoreChart` (チャート)
- `AmbiguityList` (リスト表示)

### 2. Editor (エディタ画面)

**検証項目:**
- サイドバーのカラーリング（Gray-50 背景、Purple アクティブ）
- エディタ部分の視認性（White 背景、Gray-900 テキスト）
- AI 検出結果の表示（Purple アクセント）
- FactCard の視覚デザイン

**コンポーネント:**
- `Sidebar` (ナビゲーション)
- `Editor` (テキストエリア)
- `AmbiguityDetectionPanel` (AI 検出結果)
- `FactCard` (データ表示カード)

### 3. Empty State (空状態)

**検証項目:**
- アイコン + テキストの配置
- CTA ボタンのデザイン（Purple-500 背景）
- 全体の余白感

**コンポーネント:**
- `EmptyState` (中央配置、アイコン、テキスト、ボタン)
